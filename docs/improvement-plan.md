> 목적: **외형 완성도와 검증 성숙도의 격차**를 메우는 우선순위 로드맵

## 1. 진단 요약

**잘 설계된 야심찬 프로토타입 — 단, 검증 성숙도가 외형을 못 따라간다.**

이번 세션에 발견·수정한 버그들이 전부 동일 패턴이었다:
> 기능은 만들어졌고 · 단위 테스트는 통과하고 · UI는 완성돼 보이는데 · **실제 사용 경로에서 실패**.

대표 사례:
- Strategy Coach — `and`/`or` 평가 버그로 빌트인 전략 5개 전부 백테스트 실패
- MACD 시그널선 — 초반 ~25봉 값 오류 (zero-padding 시드)
- 한국 주식 주/월봉 — tz 비교 오류로 완전 불능
- RPB — 데모 모드에서 미표시

→ 능력이 아니라 **"만들었다"와 "검증했다" 사이의 규율** 문제. 이 플랜의 최우선
목표는 그 규율(검증 레이어)을 세우는 것이다.

## 2. 우선순위 개선 항목

### P0 — 신뢰 기반 구축 (검증)

**P0-1. 통합 / E2E 테스트** · 규모: 중
- 문제: 백엔드 165개 테스트가 *통과*하는데 핵심 경로 버그가 다수 생존했다.
  단위 테스트는 파싱·계산 조각만 검증하고 "API 호출 → 실제 응답" 경로를
  검증하지 않는다. 예) `and`/`or` 버그 — 식 *파싱* 테스트만 있고 *평가*
  테스트가 없었음.
- 조치: API 엔드포인트별 통합 테스트 — 실제 요청 바디로 호출해 응답
  형태·값을 검증. 특히 `/api/strategy/backtest`(다중조건 식),
  `/api/ohlcv`(주/월봉·tz), `/api/indicators`(MACD 시그널선 값).
- 완료 기준: 2026-05-16 세션 버그를 재현하는 회귀 테스트가 전부 존재하고
  CI에서 실행된다.

**P0-2. 프론트엔드 스모크 테스트** · 규모: 소~중
- 문제: JSX 컴포넌트 테스트가 사실상 0개(테스트 파일 1개, loader 전용).
  UI 버그(겹침·잘림·반응형)가 전량 출시됐다.
- 조치: Playwright 스모크 — 5개 탭이 JS 오류 0으로 렌더되고, 핵심
  인터랙션(종목 전환·지표 토글·백테스트 실행·신호 펼치기)이 동작하는지
  확인. 데모 모드 기준이면 백엔드 의존 없이 가능.
- 완료 기준: CI에서 "5탭 × pageerror 0 + 핵심 클릭" 자동 확인.

### P1 — 기술 부채

**P1-1. 지표 이중 구현 일원화** · 규모: 중
- 문제: 지표를 `data.js`(프론트 합성, 데모)와 백엔드(pandas-ta, 라이브)가
  **각각 따로** 계산한다 → 어긋난다. (MACD 버그는 `data.js`에만 존재했고,
  볼린저밴드 표준편차도 미세하게 다름.) 유지보수 부채이자 버그 온상.
- 조치(택1): (a) 데모 모드도 백엔드 경유로 단일화, 또는 (b) `data.js`
  계산을 백엔드와 수치 일치하도록 골든값 테스트로 고정.
- 완료 기준: 동일 입력에 데모/라이브 지표값 오차 < 0.1%.

**P1-2. 빌드 파이프라인** · 규모: 중 · 우선순위 낮음
- 문제: 인-브라우저 Babel(앱 자체가 콘솔 경고). 프로덕션 부적합.
- 조치: 경량 번들러(esbuild/Vite)로 JSX 사전 컴파일.
- 비고: 프로토타입 단계에선 후순위 — 프로덕션 전환을 결정할 때 착수.
